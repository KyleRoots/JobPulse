How Job Monitoring Works

Monitor jobs in Bullhorn using custom search queries. You can create queries to track specific types of jobs:

Common Query Examples:
  • status:Open - Monitor all open jobs
  • isPublic:1 - Monitor all public jobs
  • status:Open AND isPublic:1 - Monitor open public jobs
  • clientCorporation.id:12345 - Monitor jobs for a specific client
  • title:Sales* - Monitor jobs with "Sales" in the title
  • dateAdded:[NOW-7DAYS TO *] - Monitor jobs added in the last 7 days
Query Operators:
  • AND - Both conditions must be true
  • OR - Either condition can be true
  • NOT - Exclude matching results
  • * - Wildcard for partial matches
  • [ TO ] - Range queries for dates and numbers
Pro Tip: Test your queries in Bullhorn first to ensure they return the expected results before creating a monitor.

Note: The monitor will check for changes in the jobs matching your query at the specified interval and send notifications when jobs are added, removed, or modified.
